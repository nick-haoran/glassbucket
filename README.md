# glassbucket

glassbucket is a small toolkit for researching and rebuilding replacement-slot APKs for Polytone-style Unity/CRIWARE Android packages.

The current workflow replaces an existing song slot. It keeps the original Addressables IDs, such as `TG_ALCHEMY`, `CH_ALCHEMY_BASIC`, and `AS_ALCHEMY`, then patches the related Unity `TextAsset` bundles and audio entries inside the APK.

## What is included

- `scripts/polytone_replace.py`: replace one existing song slot and optionally sign the result with `uber-apk-signer`.
- `scripts/convert_awb.py`: helper for CRI AWB/HCA extraction and rough conversion experiments.
- `scripts/compare_audio_spectrum.py`: compare decoded audio spectrum against a reference.
- `scripts/hca_key_probe.py`: probe candidate HCA keys.
- `scripts/arm64_xref.py`: lightweight IL2CPP/ARM64 string reference helper.

This repository intentionally does not include APKs, extracted assets, music files, generated bundles, keys, or signed output packages.

## Setup

```powershell
python -m pip install -r requirements.txt
powershell -ExecutionPolicy Bypass -File scripts\download_uber_apk_signer.ps1
```

`polytone_replace.py --sign` expects `tools/uber-apk-signer/uber-apk-signer-1.3.0.jar` by default.

## Replacement example

```powershell
python scripts\polytone_replace.py `
  --apk path\to\polytone.apk `
  --out out\polytone-custom-unsigned.apk `
  --slot ALCHEMY `
  --audio-acb path\to\AS_ALCHEMY.acb `
  --audio-awb path\to\AS_ALCHEMY.awb `
  --chart BASIC=path\to\basic.txt `
  --chart ADVANCED=path\to\advanced.txt `
  --chart HARD=path\to\hard.txt `
  --chart EXPERT=path\to\expert.txt `
  --patch-all-headers `
  --title "Custom Title" `
  --artist "Custom Artist" `
  --bpm 130 `
  --sign
```

The signed APK is written to `out/signed/`.

## Current limitation

The tool does not encode WAV directly into CRI ACB/AWB yet. For now, pass prebuilt replacement `.acb` and `.awb` files generated by a CRI-compatible encoder.
